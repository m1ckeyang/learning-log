阶段 0：环境搭建与心态校准（第 1 周）

学习目标

  • 注册 GitHub 账号，学会基本的 Git 操作

5. 如何重建心流状态

你说自从高考后就没有过心流。这不是你的错——高考后你的学习方式变了，从"解题"变成了"看视频"，心流自然消失了。

🧠 心流的三个前提条件（心理学家米哈里·契克森米哈赖）

1. 清晰的目标——你清楚地知道下一步要做什么
2. 即时反馈——你立刻知道自己做得对不对
3. 挑战与能力匹配——不太简单也不太难，刚刚好

高考时你有这三个条件：清晰的考纲、模拟考分数、题目难度递进。之后的学习你丢了这三个条件。这份计划就是帮你把它们找回来。

🎯

清晰目标

每个项目都有明确的"能跑了"标准。不是"学完 Python"，而是"文件整理脚本能跑了"。每天开工前写下今天要实现的一个小功能。

⚡

即时反馈

代码跑起来就是反馈。报错了就修，修好了就继续。Git commit 就是你的"得分"。每次 commit 都是一次正反馈。

📈

难度递进

项目从简单到复杂：先写 50 行脚本，再写 200 行工具，再写 500 行 API，最后写 1000+ 行的完整系统。每一步都比上一步难一点点。

🔥

番茄工作法

3 小时拆成 6 个番茄钟（25 分钟专注 + 5 分钟休息）。每个番茄钟只做一件事。不要看手机，不要开微信。手机放到另一个房间。

📝

每日日志

每天结束前花 5 分钟写：今天解决了什么问题、卡在哪里、明天要做什么。这就是你的"模拟考分数"——看得见的进步。

🔗

连续记录

GitHub 主页的绿色方块就是你的连续记录。就像打游戏保持连胜。目标是 26 周不断更。断了也没关系，第二天继续。

9. 七条原则

贴在你书桌前，每天看一遍。

1

先做再学，不做不学

不要"准备好了再开始"。写一行代码就是开始。报错了就去查，查到了就学会了。这是最高效的学习方式。

2

每天提交，雷打不动

哪怕只写了一行代码，也要 Git commit。连续记录是你的进度条，也是你给未来的自己的证据。

3

完成比完美重要

代码写得丑没关系，先跑起来。跑起来之后再优化。不要在细节上纠结到放弃。

4

一个项目一个仓库

每个项目单独建一个 GitHub 仓库。写清楚 README。这些仓库就是你的简历。

5

遇到问题先问 AI

不要在 Google 里翻了 10 页还找不到答案。直接把报错信息贴给 AI，它能 3 秒内告诉你哪里错了、怎么修。

6

手机放另一个房间

3 小时的学习时间里，手机不在身边。这是心流的物理保障。微信消息晚 3 小时回，天塌不下来。

7

记住你为什么开始

还房贷、结婚、改变人生。每次想放弃的时候，想想这三个词。然后打开电脑，写一行代码。
